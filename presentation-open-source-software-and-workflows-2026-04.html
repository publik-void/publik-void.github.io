Open Source

What does “Open Source” mean?

Open Source means:

  • The instructions for making something are public
  • Anyone is allowed to use, study, modify, and share them

Examples:

  • Furniture, bicycles, farm tools, prostheses, …
  • Cosmetics, soaps, food products, …
  • Computer hardware

Antonyms:

  • Proprietary
  • Closed source

Free and open source software (FOSS)

The Free Software Foundation defines four essential freedoms:

  • To run the program
  • To study and change the program in source code form
  • To redistribute exact copies
  • To distribute modified versions

Interoperability

  • Connecting closed tools is permission-based.
  • Connecting open tools is possibility-based.

Examples of vendor lock-in:

  • Poor integration of iCloud services on non-Apple devices
  • Microsoft bundling Teams with Office
  • Kindle e-books not readable outside Kindle
  • HP printers refusing third-party cartridges
  • …
  • EU Digital Markets Act

Embrace, extend, extinguish

Right to repair

  • Closed tools' lifetime is vendor-controlled.
  • Open tools are fixable.

Examples of anti-repair design:

  • Dealer-only software locks on John Deere tractors
  • No disassembly of Microsoft Surface laptops
  • Samsung washing machines requiring whole-tub replacement
  • …
  • EU Directive on repair of goods

Reproducibility & replicability

  • Reproducibility: Same data, same methods
  • Replicability: New data, same question

“More than 70% of researchers have tried and failed to reproduce another scientist's experiments, and more than half have failed to reproduce their own experiments.”

— Monya Baker, Nature

  • Closed tools are black boxes, hiding methodological details.
  • Open tools expose the full workflow, without license barriers.

Downsides of open-source software

  • Funding problems
  • Fragmentation
  • No guaranteed accountability
  • Licensing complexity

Workflows

What is a scientific workflow?

  • A defined sequence of repeatable steps
  • Each step transforms data or materials
  • Outputs from one step feed into further steps
  • Makes analyses repeatable
  • Makes results reproducible
  • Facilitates checking, tracing, reusing, and sharing

Best practices for reproducible workflows

  • Name things clearly and consistently
  • Write documentation:
    • “How must this step be carried out exactly?”
    • “What does this variable measure?”
    • “What is this script for?”
  • Reduce hidden dependencies between steps
  • Automate repetitive work if possible
  • Prefer open-source tools where practical
  • Choose suitable, open data formats
  • Keep data traceable and backed up

Version control

tracks changes to files over time:

  • Trace text, figures, datasets, code
  • See what changed, when, and why
  • Recover earlier versions safely
  • Reduce confusion from many copies
  • Collaborate without overwriting each other's work
  • Keep an online project backup
